1. Entendimento do Negócio¶

Imagine uma empresa que quer saber sobre a repercusso de uma campanha de marketing que foi lanada no mercado. possvel usar as redes sociais para buscar informaes, mas como saber se esto falando bem ou mal dessa campanha? Ler, entender e tirar uma concluso de milhares de comentrios de usurios pode ser uma tarefa invivel.

nesse cenrio que o objetivo desse projeto se enquadra. Iremos desenvolver um modelo de aprendizado de mquina capaz de classificar tweets sobre um determinado tema em positivo ou negativo. Aqui, no iremos avaliar nenhuma campanha, produto ou servio de uma empresa especfica, em vez disso, utilizaremos tweets relacionados a Black Friday, um tema mais "genrico" que no ir expor nenhuma empresa.

O grande desafio desse projeto se d porque no existe um conjunto de dados de tweets pr-etiquetados para treinar o modelo, ento, para a fase de treinamento utilizaremos outros dados de contexto parecido, o de avaliaes do grupo B2W Digital. Este grupo responsvel pelas lojas Submarino, Shoptime e Americanas.

Outro desafio desse projeto em relao a linguagem das mensagens publicadas no Twitter que, em geral, contm grias, ironias, e caracteres especiais, ento, teremos de realizar o pr-processamento nesses dados, para que o modelo consiga classificar textos que "nunca viu antes" de forma satisfatria. Para isso, ser utilizado o algoritmo Multinomial Naive Bayes, que um modelo probabilstico baseado na frequncia de um termo e excelente para classificao de documentos.

1.2 Dicionário de Dados¶

Como descrito acima, iremos utilizar o conjunto de dados do grupo B2W Digital para o treinamento do modelo.

Esse conjunto est dividido em dois arquivos, "train.csv" e "test.csv", porm, iremos utilizar apenas o conjunto de treino e os seguintes atributos:

  • review_text: texto principal da avaliao.
  • rating: nota atribuda a avaliao, de 1 a 5.

1.3 Estratégia da Solução¶

Como estratgia para a soluo do projeto, definimos as seguintes etapas:

  • 1. Entendimento do Negcio: nessa etapa inicial, realizamos uma breve introduo sobre o tema do projeto e definimos nossos objetivos.
  • 2. Entendimento dos Dados: nessa etapa, definimos o conjunto de dados que iremos utilizar e realizamos um tratamento a fim de verificar possveis inconsistncias nos dados.
  • 3. Pr-Processamento dos Dados: o principal objetivo nessa etapa realizar a preparao dos dados para a modelagem. Entre as prticas abordadas nessa sesso esto: balanceamento de classes, limpeza dos dados, lematizao e transformao dos dados em vetores numricos.
  • 4. Modelagem Preditiva: aqui, iremos criar o modelo preditivo utilizando o algoritmo Naive Bayes. Como mtricas de avaliao, verificaremos a acurcia, recall, F1 Score e tambm a matriz de confuso.
  • 5. Similaridade de Dados: esta etapa foi inserida como uma anlise adicional que tem como objetivo verificar as similaridades entre as palavras mais frequentes. Para a realizao dessa tarefa utilizamos o algoritmo Word2Vec.
  • 6. Classificando Tweets: esta etapa se refere ao objetivo principal do projeto. Com o modelo treinado, iremos realizar a coleta de uma nova base de dados diretamente do Twitter, utilizando a prpria API da rede social. Aps isso, iremos apresentar esses dados ao modelo preditivo para analisarmos a sua performance.
  • 7. Concluses Finais: por fim, aqui, iremos deixar nossas concluses finais do projeto.

3.2 Limpeza dos Dados¶

Agora, vamos tratar a varivel review_text.

Teremos como objetivo:

  • Remover palavras irrelevantes e acentuaes.
  • Substituir letras maisculas por minsculas.
  • Aplicar a lematizao.
  • Converter textos para vetores numricos.
In [22]:
# Definindo a classe.

# Carregando o pipeline em portugus.
nlp = spacy.load('pt_core_news_sm')

# Criando a classe.
class DataPrep:
    
    # Funo para remover acentos e cedilha.    
    def rm_accents(self, texto) -> list:
        fixed = list()
        for linha in texto:
            unidecoded_text = unidecode(linha)
            fixed.append(unidecoded_text)
        return fixed
    
    # Definindo a funo de stopwords.    
    def remove_stopwords(self, texto):
        stop_words = [word for word in nlp.Defaults.stop_words]
        remover = ['lojas', 'americanas', 'americana', 'submarino', 'shoptime']
        stop_words.extend(remover)
        texto_limpo = " ".join([i for i in texto if i not in set(stop_words)])
        return texto_limpo
    
    # Funo para aplicar a limpeza no corpus (remove stopwords, acentos e letras maisculas).
    def clean_text(self, texto):
        df_corpus = []
        for i in range(len(texto)):
            df_c = re.sub('[^A-Za-z]', ' ', texto[i]).lower().split()
            df_corpus.append(df_c)
        df_corpus = pd.Series(df_corpus).apply(lambda x: ' '.join([w for w in x if len(w) > 2]))
        corpus = [self.remove_stopwords(r.split()) for r in df_corpus]
        return corpus
    
    # Funes para aplicar a lematizao.
    def lemmatization(self, texto):
        global nlp        
        output = []
        for sent in texto:
            doc = nlp(" ".join(sent)) 
            output.append([token.lemma_ for token in doc])
        return output
    
    def lemmatize(self, texto):
        token = self.lemmatization(pd.Series(self.clean_text(texto)).apply(lambda x: x.split()))
        token_lemma = []
        for i in range(len(token)):
            token_lemma.append(' '.join(token[i]))
        return token_lemma

3.2.1 Lematização¶

A lematizao diz respeito ao processo de agrupar as formas flexionadas de uma palavra para que possam ser analisadas como um nico item, identificado pelo "lema" da palavra.

Por exemplo, as palavras gato, gata, gatos e gatas so todas formas do mesmo lema: gato.

Igualmente, as palavras tiver, tenho, tinha, tem so formas do mesmo lema: ter.

A vantagem de aplicar a lematizao clara: reduo de vocabulrio e abstrao de significado.

Existem pacotes com funes prontas para realizar a lematizao, mas aqui, vamos utilizar a funo que definimos na classe DataPrep.

Iremos salvar o resultado desse tratamento na varivel corpus.

3.2.2 Conversão dos Textos para Vetores Numéricos¶

Para realizar essa tarefa, iremos utilizar uma tcnica chamada TF-IDF.

TF-IDF de forma traduzida, significa "Frequncia do Termo - Frequncia Inversa de Documentos".

Sabemos que os computadores "entendem" apenas dados numricos, ento, por esse motivo, vetorizamos todo o texto para que o computador possa compreend-lo.

Geralmente calculamos um peso para cada palavra, e esse peso significa a importncia da palavra dentro do corpus.

Para essa tarefa, utilizaremos a classe *TfidfVectorizer()* presente no Scikit-Learn e definiremos alguns hiperparmetros:

  • min_df: desconsidera termos que aparecem menos que "X" vezes.
  • max_df: usado para excluir termos que so muito frequentes e que provavelmente no ajudaro na previso do modelo.
  • ngram_range: define o intervalo de valor desejado (unigramas, bigramas, trigramas, n-grama).
  • max_features: nmero de termos mais frequentes para se considerar.

Interpretao das mtricas:

  • Precision: dentre todas as previses realizadas para uma classe, quantas o modelo acertou.

5. Similaridade de Dados¶

Deixando o objetivo principal do projeto de lado, nessa etapa queremos encontrar as similaridades entre as palavras mais frequentes do corpus.

Para isso, utilizaremos o Word2Vec, um dos algoritmos mais utilizados para esse tipo de tarefa.

Esse modelo usa como entrada um corpus de texto e produz um espao vetorial, tipicamente com vrias centenas de dimenses, com cada palavra exclusiva no corpus sendo atribuda a um vetor correspondente no espao. Os vetores de palavras so posicionados no espao vetorial, de modo que as palavras que compartilham contextos comuns no corpus sejam localizadas prximas umas das outras no espao.

O Word2Vec espera receber como entrada um corpus com uma "sequncia de frases", onde cada frase uma lista de palavras.

Sendo assim, precisaremos realizar um pequeno tratamento nos dados.

In [43]:
# Preparando o corpus.

O cdigo acima cria o objeto sent contendo a sequncia de frases.

Agora, precisamos modelar os bigramas.

Os bigramas so duas palavras que ocorrem frequentemente juntas.

O pacote gensim possui um modelo chamado Phrases que detecta automaticamente os bigramas presentes em nosso conjunto.

Iremos definir dois parmetros desse modelo:

  • min_count: quantas vezes uma palavra precisa aparecer, no mnimo, para ser considerada.

  • threshold: representa um limite para formar as frases (mais alto significa menos frases).

Mais detalhes sobre os hiperparmetros desse algoritmo esto disponveis nesse link.

In [44]:
# Instanciando o modelo.

                  min_count = 1, 
                  threshold = 2)

In [45]:

Tambm definiremos alguns valores para os hiperparmetros:

  • min_count: quantas vezes uma palavra precisa aparecer, no mnimo, para ser considerada.
  • window: tamanho mximo das frases que o modelo vai analisar.
  • sample: limite para configurar quais palavras mais frequentes sero descartadas.

Para mais detalhes sobre os hiperparmetros desse algoritmo acesse este link.

# Criando a funo.
def rm_spam(df, col):
    index_list = list()
    for i in range(df.shape[0]):
        match = re.search(r'\Shttp|\bhttp', df[col][i])
        if match:
            index_list.append(i)
        else:
            continue
    df_filtered = df.drop(index_list, axis = 0).reset_index(drop = True)
    return df_filtered

7. Conclusões Finais¶

O maior desafio desse projeto estava em no possuir um conjunto de dados pr-etiquetados para o treinamento do modelo.

Por conta disso, utilizamos um conjunto com dados similares ao tema abordado.

Durante a fase de preparao dos dados, aplicamos tcnicas amplamente utilizadas em PLN e que foram cruciais para o modelo conseguir interpretar as informaes e realizar as classificaes em um conjunto de dados relativamente diferente do que foi utilizado para seu treinamento.

Em relao ao desempenho do modelo treinado com o algoritmo Naive Bayes, conseguimos um resultado bastante satisfatrio.

Podemos perceber que para alguns tweets, o modelo teve maior dificuldade para realizar as classificaes, e por esse motivo, consideramos apenas as classificaes mais assertivas.

Apesar desse projeto ter uma complexidade de nvel bsico em PLN, poderamos utilizar as mesmas abordagens vistas aqui para treinar um modelo capaz de classificar tweets referentes a qualquer tema, e assim, auxiliar as reas de interesse a tomar as melhores decises para a empresa.
